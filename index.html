Proyecto web con Git y GitHub

Pagina web colaborativa con flujo Git completo

Sitio creado para practicar un flujo de desarrollo en equipo usando Git, GitHub, ramas de trabajo, pull requests y resolucion de conflictos.

Ver proyecto

Objetivo

Aplicar un flujo de trabajo real

El objetivo del proyecto es construir una pagina web sencilla mientras el equipo practica la creacion de ramas, commits descriptivos, revision de codigo mediante pull requests y solucion de conflictos antes de integrar cambios en la rama principal.

Equipo

Integrantes y roles

Lider

Coordina el avance, organiza tareas y valida que se cumplan los requisitos.

Documentador

Redacta el README, registra decisiones y describe el flujo usado por el equipo.

Integrador

Gestiona ramas, pull requests, revisiones y fusiona cambios aprobados.

Disenador

Define la apariencia visual, colores, espaciado y experiencia de usuario.

Flujo

Trabajo con ramas

  1. La rama principal es main.
  2. Los cambios de desarrollo se preparan en dev.
  3. Los ajustes visuales se trabajan en diseno.
  4. Antes de integrar cambios, se abre un pull request y otro integrante revisa el codigo.
  5. Si hay conflictos, se resuelven localmente y se registra el resultado con un commit claro.

Proyecto academico para practicar Git, GitHub y colaboracion en equipo.
